# AscendC Direct Launch — Complete Compilable Project Template

This page closes the gap between **API snippets** and a **working build**. It documents the end-to-end project layout used by MultiKernelBench's `ascendc_direct_launch` backend: bisheng-compiled device kernels, pybind11 host binding, and a thin Python `ModelNew.forward()` that only dispatches to the custom extension.

> **Official basis**: CANN 9.0 [PyTorch Framework Adaptation](https://www.hiascend.com/document/detail/zh/canncommercial/900/programug/Ascendcopdevg/atlas_ascendc_10_0057.html) (doc-ascendc-pytorch-framework-adaptation) documents the same Kernel直调 pattern with `getCurrentNPUStream` + `<<<>>>` launch. MKB auto-generates CMake on top of this pattern.

> **Core insight from agent sessions**: rounds 1–2 failed because solvers could assemble AscendC API fragments but not the full CMake + include path + kernel/binding split. Copy this skeleton first; optimize the kernel second.

## torch.library vs Pybind (Official Guidance)

| Aspect | torch.library | Pybind (`PYBIND11_MODULE`) |
|--------|---------------|----------------------------|
| Python call | `torch.ops.ns.op(x, y)` | `import mod; mod.op(x, y)` |
| Schema / graph tracing | Yes | **No** |
| torch.compile | Supported | **Not supported** |
| MKB ascendc_direct_launch | Works if ModelNew imports module | **Recommended** — matches eval flow |
| Official recommendation | Production default | Quick binding only |

Source: doc-ascendc-pytorch-framework-adaptation — Huawei explicitly states Pybind lacks schema and cannot support torch.compile.

## Project Layout

```
submission/
├── ModelNew.py              # Python entry — ONLY calls custom extension
└── kernel/
    ├── add_launch.h         # Host-side launch declarations + tiling helper
    ├── add_kernel.cpp       # AscendC kernel (bisheng -xasc)
    ├── pybind11.cpp         # torch_npu binding (g++ link)
    └── build/               # Generated by CMake (extension .so lands here)
```

The evaluator writes all files from a JSON `sources[]` array, auto-generates `CMakeLists.txt`, runs `cmake` + `cmake --build`, then `exec()`s `ModelNew.py`.

## Required Include Paths (CANN + torch_npu)

The auto-generated CMakeLists adds these CANN paths (via `ASCEND_CANN_PACKAGE_PATH`):

| Path suffix | Purpose |
|-------------|---------|
| `include/` | ACL / runtime headers |
| `include/experiment/runtime` | Runtime experiment APIs |
| `compiler/tikcpp/include` | TIK C++ helpers |
| `compiler/ascendc/include/basic_api/impl` | AscendC basic API impl |
| `compiler/ascendc/include/basic_api/interface` | AscendC basic API interface |
| `compiler/ascendc/include/highlevel_api/impl` | Matmul / high-level APIs |
| `compiler/ascendc/include/highlevel_api/tiling` | `MultiCoreMatmulTiling` |
| `compiler/ascendc/impl/aicore/basic_api` | AICore basic impl |

Plus runtime-discovered paths:

- `torch/include`, `torch/include/torch/csrc/api/include`
- `torch_npu/include`
- `pybind11` includes from `python3 -m pybind11 --includes`

**Common failure**: missing `platform/platform_ascendc.h` → add `compiler/ascendc/include/highlevel_api/tiling` and ensure `#include "platform/platform_ascendc.h"` in host-side tiling code.

## kernel/add_launch.h — Launch Declarations

```cpp
#ifndef ADD_LAUNCH_H
#define ADD_LAUNCH_H
#include <cstdint>
#include <tuple>

#ifndef GM_ADDR
#define GM_ADDR void*
#endif

std::tuple<int64_t, int64_t, int64_t> calc_add_tiling_params(int64_t totalLength);

extern "C" {
void launch_add_kernel_half(GM_ADDR x, GM_ADDR y, GM_ADDR z,
    int64_t totalLength, int64_t numBlocks, int64_t blockLength,
    uint32_t tileSize, void* stream);
}
#endif
```

**Rules**:
- `extern "C"` launch wrappers live in the same `.cpp` as the kernel OR a separate host `.cpp` — never inside `__global__ __aicore__` functions.
- `GM_ADDR` must be defined before use (CANN typedef; fallback to `void*`).

## kernel/add_kernel.cpp — Device Kernel + Host Launch

```cpp
#include "kernel_operator.h"
#include "platform/platform_ascendc.h"

constexpr static int64_t PIPELINE_DEPTH = 2;

template <typename T>
__global__ __aicore__ void add_kernel(GM_ADDR x, GM_ADDR y, GM_ADDR z,
    int64_t totalLength, int64_t blockLength, uint32_t tileSize)
{
    AscendC::TPipe pipe;
    AscendC::GlobalTensor<T> xGm, yGm, zGm;
    // ... InitBuffer, DataCopyPad, Add, EnQue/DeQue ...
    // Use AscendC::GetBlockIdx() for multi-core block offset
}

std::tuple<int64_t, int64_t, int64_t> calc_add_tiling_params(int64_t totalLength)
{
    auto* platform = platform_ascendc::PlatformAscendCManager::GetInstance();
    uint64_t ubSize = 0;
    platform->GetCoreMemSize(platform_ascendc::CoreMemType::UB, ubSize);
    int64_t coreNum = platform->GetCoreNumAiv();
    // ... compute numBlocks, blockLength, tileSize ...
    return {numBlocks, blockLength, static_cast<int64_t>(tileSize)};
}

extern "C" void launch_add_kernel_half(/* ... */ void* stream)
{
    add_kernel<half><<<numBlocks, nullptr, stream>>>(
        x, y, z, totalLength, blockLength, tileSize);
}
```

**Boilerplate checklist**:
- Use `AscendC::` prefix OR `using namespace AscendC;` — but see pattern-ascendc-compile-troubleshooting for `matmul` namespace conflicts.
- Kernel entry: `__global__ __aicore__ void name(...)`.
- Launch syntax: `kernel<<<blockDim, nullptr, stream>>>(args...)`.
- Tiling on host side uses `platform_ascendc::PlatformAscendCManager`.

## kernel/pybind11.cpp — torch_npu Binding

Official CANN add example (doc-ascendc-pytorch-framework-adaptation):

```cpp
#include <pybind11/pybind11.h>
#include <torch/extension.h>
#include "torch_npu/csrc/core/npu/NPUStream.h"
#include "add_launch.h"
```

namespace my_ops {

torch::Tensor add(const torch::Tensor& x, const torch::Tensor& y)
{
    TORCH_CHECK(x.sizes() == y.sizes(), "shape mismatch");
    const c10::OptionalDeviceGuard guard(x.device());
    auto z = torch::empty_like(x);
    auto stream = c10_npu::getCurrentNPUStream().stream(false);

    int64_t totalLength = x.numel();
    int64_t numBlocks, blockLength, tileSize;
    std::tie(numBlocks, blockLength, tileSize) = calc_add_tiling_params(totalLength);

    auto xPtr = reinterpret_cast<GM_ADDR>(x.data_ptr());
    auto yPtr = reinterpret_cast<GM_ADDR>(y.data_ptr());
    auto zPtr = reinterpret_cast<GM_ADDR>(z.data_ptr());

    launch_add_kernel_half(xPtr, yPtr, zPtr, totalLength,
        numBlocks, blockLength, static_cast<uint32_t>(tileSize), stream);
    return z;
}

} // namespace

PYBIND11_MODULE(benchmark_ops, m)
{
    m.def("add", &my_ops::add, "AscendC add");
}
```

See **lang-torch-npu-cpp-api** for full header list and stream semantics.

## ModelNew.py — Thin Python Wrapper

```python
import sys
from pathlib import Path
import torch
import torch.nn as nn

_KERNEL_BUILD = Path(__file__).resolve().parent / "kernel" / "build"
if _KERNEL_BUILD.is_dir() and str(_KERNEL_BUILD) not in sys.path:
    sys.path.insert(0, str(_KERNEL_BUILD))

import benchmark_ops as _ops

class ModelNew(nn.Module):
    def forward(self, a: torch.Tensor, b: torch.Tensor) -> torch.Tensor:
        return _ops.add(a, b)

def get_init_inputs():
    return []
```

**Critical**: `forward()` must contain **zero** PyTorch compute ops — only extension calls. The AST cheating detector rejects `torch.add`, tensor arithmetic, `F.*`, etc.

## JSON Submission Format

Wrap the project as JSON (first code block in model output):

```json
{
  "name": "my_add_kernel",
  "format_version": "0.1",
  "entry": { "model": "ModelNew.py::ModelNew" },
  "build": {
    "type": "ascendc_direct_launch",
    "module_name": "benchmark_ops",
    "kernel_sources": ["kernel/add_kernel.cpp"],
    "binding_sources": ["kernel/pybind11.cpp"],
    "include_dirs": ["kernel"]
  },
  "sources": [
    { "path": "ModelNew.py", "content": "..." },
    { "path": "kernel/add_launch.h", "content": "..." },
    { "path": "kernel/add_kernel.cpp", "content": "..." },
    { "path": "kernel/pybind11.cpp", "content": "..." }
  ]
}
```

| Field | Required | Notes |
|-------|----------|-------|
| `entry.model` | yes | `"File.py::ClassName"` |
| `build.kernel_sources` | recommended | `.cpp` compiled with bisheng; inferred if omitted |
| `build.binding_sources` | recommended | `.cpp` with `PYBIND11_MODULE`; inferred if omitted |
| `build.module_name` | optional | Must match `PYBIND11_MODULE(name, m)` |
| `build.include_dirs` | optional | Defaults to `.` + parent dirs of sources |
| `build.build_dir` | optional | Default: `{binding_parent}/build` |

## CMake Build Flow (Auto-Generated)

The backend generates CMake that:

1. Finds `bisheng` under `$ASCEND_CANN_PACKAGE_PATH/bin/bisheng`
2. Compiles each `kernel_sources` entry: `bisheng --npu-arch=dav-2201 -xasc -std=c++17 -fPIC -c ...`
3. Links `binding_sources` + kernel `.o` files into `pybind11_lib` SHARED
4. Links: `torch`, `torch_cpu`, `torch_npu`, `ascendcl`, `platform`, `register`, `tiling_api`, `ascendc_runtime`, `runtime`, `profapi`

Environment variables:

```bash
export ASCEND_HOME_PATH=/path/to/ascend-toolkit/latest
export SOC_VERSION=Ascend910B2   # passed to cmake -DSOC_VERSION=...
```

## Common Compile Errors (Quick Reference)

| Error | Fix |
|-------|-----|
| `bisheng compiler does not exist` | Set `ASCEND_CANN_PACKAGE_PATH` / `ASCEND_HOME_PATH` |
| `kernel_operator.h: No such file` | Missing CANN include path; use auto-generated CMake |
| `platform_ascendc.h: No such file` | Add highlevel_api/tiling include |
| `NPUStream.h: No such file` | Add `torch_npu/include` |
| `PYBIND11_MODULE` not found | Run `python3 -m pybind11 --includes`; link Python3 |
| `reference to 'matmul' is ambiguous` | Don't use bare `using namespace AscendC` with Catlass matmul |
| `LocalTensor<T>` mismatch | Ensure same `T` in AllocTensor/DeQue template arg |
| Link error `undefined reference to launch_*` | Launch wrapper must be `extern "C"` in kernel .cpp |

Full diagnostic playbook: **pattern-ascendc-compile-troubleshooting**.

## Migration Path: Snippet → Working Project

1. Start from `prompts/ascendc_direct_launch_model_add.json` in MultiKernelBench (verified runnable).
2. Replace `add_kernel.cpp` logic with your operator; keep launch wrapper pattern.
3. Extend `pybind11.cpp` with dtype dispatch (`float`/`half`/`bfloat16`).
4. Update `ModelNew.forward()` signature to match benchmark reference.
5. Run locally: `python3 eval_single_runner.py -i submission.json -o ffn -l ascendc_direct_launch -r result.json`
