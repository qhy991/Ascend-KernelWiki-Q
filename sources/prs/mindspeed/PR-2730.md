# MindSpeed — MindSpore MoE All-to-All Compute/Communication Overlap

## What Changed

MindSpeed PR #2730 adds or updates MindSpore MoE all-to-all overlap behavior. The verified diff touches three files:

- `mindspeed/mindspore/core/transformer/moe/grouped_mlp_with_comp_and_comm_overlap_all2all.py`
- `mindspeed/mindspore/core/transformer/moe/legacy_a2a_token_dispatcher.py`
- `mindspeed/mindspore/core/transformer/moe/moe_layer_overlap_all2all.py`

The change reorganizes token dispatch, grouped MLP execution, and asynchronous all-to-all communication so compute and communication can overlap in the MoE layer.

## Why It Matters

MoE training has a recurring bottleneck: token all-to-all can serialize with expert MLP compute. Overlap paths attempt to dispatch tokens, run grouped expert MLP, and exchange outputs in a schedule that hides communication behind compute. This PR is valuable because it changes the actual MindSpore training path rather than only adding scripts.

## Cost Model

| Without overlap | With overlap |
| --- | --- |
| token all-to-all waits before/after expert compute | communication is scheduled around GroupedMLP work |
| easier control flow | more complex saved tensors and backward ordering |
| lower risk | higher throughput if comm latency is hidden |

The trade-off is complexity: autograd saved tensors, token permutation, and async all-to-all ordering must remain consistent in forward and backward.
