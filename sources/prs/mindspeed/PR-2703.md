# MindSpeed — Fused MoE Token Permute and Unpermute on NPU

## What Changed

MindSpeed PR #2703 adds a fused MoE token permutation feature. The diff creates `mindspeed/core/fusions/fused_moe_permute.py` and `mindspeed/features_manager/fusions/fused_moe_permute.py`, and patches Transformer Engine permutation behavior in `mindspeed/te/pytorch/permutation.py`.

The new wrapper routes fused mode to `torch_npu` operators such as `npu_moe_token_unpermute_with_routing_map`.

## Why It Matters

MoE token dispatch requires permuting tokens into expert order and unpermuting results back to original token order. These are memory-movement-heavy operations. Fusing token permute/unpermute on NPU reduces Python/framework overhead and can avoid intermediate layout churn in the dispatch path.

- Benefits are strongest when token permutation is a visible memory/host overhead.
