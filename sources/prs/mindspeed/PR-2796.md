# MindSpeed — MLA Supports Ring Attention and Tensor Parallel Context Parallel

## What Changed

MindSpeed PR #2796 adds MLA support for ring attention and tensor parallel context-parallel training. The verified Gitee diff changes seven files under context parallel, flash attention reset-mask adapters, multi-head latent attention adapters, and tests.

The core change is that MLA K/V tensors may have different last dimensions. The previous stacked `[2, ...]` KV assumption does not hold. The PR represents K/V as a list or tuple, flattens and concatenates them for P2P transfer, then slices and views them back into separate K/V tensors after communication.

## Why It Matters

Ring context parallelism passes K/V blocks around ranks. Standard attention can often stack K and V into one uniform tensor. MLA breaks that assumption because K and V can have different latent dimensions. This PR is important because it identifies the correct abstraction boundary: communication payloads must support heterogeneous K/V shapes while still using a single ring P2P exchange.

## Mechanism

The RingP2P path is generalized:

- Detect MLA when K/V final dimensions differ.
- Represent K/V as `[k, v]` instead of stacked `[2, ...]` tensor.
- Validate list/tuple length and shape expectations.
- Flatten each tensor with `view(-1)`, concatenate, and send through P2P.
- After receive completion, slice the flat buffer and `view` back into K/V shapes.
- Keep TP/SP gather/scatter logic aligned with MLA attention adapters.

## Trade-offs and Pitfalls

- **Uneven shapes remain limited.** The agent found the PR explicitly does not support uneven MLA CP shapes.
- **Receive buffer is view-based.** Consumers must wait for communication before using restored K/V views.
- **This is not a single AscendC kernel.** It is training-side communication adaptation for MLA over ring context parallelism.
- **TP/SP and CP interact.** Sequence and hidden dimensions must be gathered/scattered in the same semantic order as ring P2P.
