# fused_rope_qk_mqa optimize & fix glm5 bf16 no bias

## Summary
This PR optimizes the Triton kernel `fused_rope_qk_mqa_kernel` for applying Rotary Position Embeddings (RoPE) to Q and K in Multi-Query Attention (MQA) topologies.

## Optimizations
1. **Shared Cos/Sin Loading**: Loads the cosine and sine components once per position `pid_t` and shares them across all `Hq` heads.
2. **Batched Q Loading**: Eliminates the individual head loop by using vectorized array indexing (`head_offs[:, None]`) to load all heads simultaneously.
