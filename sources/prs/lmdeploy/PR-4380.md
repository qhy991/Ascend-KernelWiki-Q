# Adapt for s1-pro dp*tp+ep on Ascend

## Summary
To scale massive Mixture-of-Experts (MoE) models like DeepSeek-V3 / S1-Pro on Huawei Ascend hardware, purely relying on Tensor Parallelism (TP) hits network bandwidth bottlenecks across HCCL communication rings. This PR introduces a hybrid parallelism strategy (`DP * TP + EP`) for LMDeploy's Ascend backend.

## Implementation Details
The architecture combines:
1. **Data Parallelism (DP)**: Batches are split across disjoint nodes.
2. **Tensor Parallelism (TP)**: Dense attention and MLP computations are sharded across NPU cards within the same high-bandwidth chassis (e.g., an 8-NPU box).
3. **Expert Parallelism (EP)**: MoE routed experts are distributed across nodes.

By mapping the HCCL communication groups properly, the `AllToAll` collective required for Expert Parallel routing accurately intersects with the `AllReduce` required for Tensor Parallelism, without deadlocking or producing dimension mismatches on the Ascend NPU platform.
