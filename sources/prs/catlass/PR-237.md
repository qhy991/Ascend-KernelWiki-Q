# CATLASS — Online Softmax RowNum=1 Wait Boundary Fix

## What Changed

Gitee PR !237 fixes the CATLASS online-softmax no-mask epilogue for a boundary case where `rowNum = 1`. Agent-verified diff evidence shows it is a focused single-file change in:

- `include/catlass/epilogue/block/block_epilogue_online_softmax_no_mask.hpp`

## Why It Matters

Online softmax overlaps score production, row-wise reduction, and later probability/value work. A wait that is valid for multi-row progress can be wrong or unnecessary when only one row is active. This PR is valuable because it isolates a classic pipeline-boundary bug: synchronization policy depends on the active row/block shape, not only on the nominal kernel template.

## Mechanism

The bug sits in the epilogue, not in the Cube MMAD. The fix adjusts the row loop / wait boundary so the online-softmax epilogue does not wait on QK progress in a shape where that dependency is not present in the same way.

## Trade-offs and Pitfalls

- **Synchronization is shape-dependent.** A row-count tail case can invalidate a wait rule that passed normal tiles.
- **Small epilogue fixes can unblock attention correctness.** The matrix multiply path can be correct while the reduction/wait path is wrong.
- **Tail shapes need targeted tests.** Include row count 1, last block, and nonuniform row-count cases when testing streaming attention.
