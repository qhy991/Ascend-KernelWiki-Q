## Why It Matters

Attention backward is not a single GEMM. It is a multi-stage pipeline that recomputes or consumes softmax statistics, applies masks, computes softmax gradients, and launches several matrix multiplications. This PR is high-value because it shows how CATLASS decomposes attention backward into reusable Cube and Vector block components.

## Execution Path

The implementation splits attention backward into multiple phases:

1. Cube stage for `Q * K^T` and `dOut * V^T`-style products.
2. Vector epilogue/preprocessing for mask, row max/sum, and softmax-gradient terms.
3. Cube stage for `dQ = dS * K`.
4. Cube stage for `dK = dS^T * Q` and `dV = P^T * dOut`.
5. Workspace offset management and cross-core synchronization to coordinate intermediate tensors.

## Mechanism

The CATLASS design uses specialized block MMAD components:

- `block_mmad_fag_cube1.hpp`
- `block_mmad_fag_cube2.hpp`
- `block_mmad_fag_cube3.hpp`

Vector epilogue headers implement the non-GEMM pieces around softmax grad and postprocessing. GM→L1 and L1→L0 copy primitives are adjusted so the Cube stages can feed from the correct tiled layout.

## Cost Model

The performance target is not merely “fewer launches.” It is structured reuse:

| Component | Purpose |
| --- | --- |
| Cube MMAD stages | handle attention backward GEMMs at matrix throughput |
| Vector epilogues | handle mask, softmax grad, and postprocessing |
| Workspace layout | keep recomputed/intermediate stats addressable across phases |
| Cross-core sync | coordinate multi-core stages that share intermediate state |

## Trade-offs and Pitfalls

- **Backward attention has more state than forward attention.** Row max/sum, probabilities, masks, and gradients must stay consistent.
- **Workspace offsets are part of correctness.** Wrong offset planning corrupts later gradient stages.
- **Cube/Vector boundaries matter.** Softmax grad is vector-heavy; GEMM stages are Cube-heavy.
- **TND layout assumptions must match the example.** Porting to another layout needs revisiting copy and epilogue policies.
