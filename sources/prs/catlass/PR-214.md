# CATLASS — Grouped Matmul Slice-K Ki=0 Output Clear

## What Changed

Gitee PR !214 fixes the `grouped_matmul_slice_k` example and kernel path for a `Ki = 0` boundary. Agent-verified diff evidence shows two changed files:

- `examples/05_grouped_matmul_slice_k/grouped_matmul_slice_k.cpp`
- `include/catlass/gemm/kernel/grouped_matmul_slice_k.hpp`

## Why It Matters

Grouped GEMM often has uneven expert/group shapes. In a Slice-K path, a group with empty K dimension may not execute normal Cube accumulation, but the output tensor still has observable semantics. If the kernel skips computation without clearing output, stale data can leak into the result.

## Mechanism

The fix treats `Ki = 0` as an explicit output-initialization case. Rather than relying on the normal matmul accumulation path, the grouped kernel clears the corresponding output region so empty groups produce deterministic zero output.

## Trade-offs and Pitfalls

- **Empty work is still semantic work.** A group that has no K tiles may still need output initialization.
- **Split-K paths magnify tail cases.** Per-group K can differ, so a global shape check is not enough.
- **Output clearing has cost.** The branch should be limited to empty-K groups, not inserted into the hot accumulation path.
