## Why It Matters

Flash Attention inference is a good CATLASS architecture slice because it combines Cube-heavy QK/PV matrix products with Vector-heavy online softmax and output rescaling. The PR shows how CATLASS factors those responsibilities into block MMAD policies, epilogue policies, and host tiling code rather than a monolithic kernel.

## Execution Path

The example expresses the forward attention pipeline as:

1. Host code prepares FAI shapes, strides, and tiling metadata.
2. QK block MMAD computes score tiles.
3. Online softmax epilogue maintains row-wise max and sum state across streamed K/V blocks.
4. PV block MMAD consumes normalized probability tiles and V tiles.
5. Output rescale epilogue writes final O without requiring a full score matrix in global memory.

## Mechanism

The key mechanism is stateful streaming. The kernel does not materialize all attention scores. It carries per-row softmax state through block epilogues, while Cube stages process tiled matrix products. This makes workspace and epilogue state part of correctness, not just performance.

## Trade-offs and Pitfalls

- **Online-softmax state is live across blocks.** Row max/sum and rescale state must match the K/V tile loop order.
- **QK and PV have different dataflow needs.** Splitting them into dedicated block MMAD components makes policy choices explicit.
- **Host tiling is part of the kernel.** `fai_tiling.cpp` determines whether block shapes fit the streaming pipeline.
- **No-mask forward is not general masked attention.** Masked, causal, paged, or variable-length forms need extra epilogue and address policy checks.
