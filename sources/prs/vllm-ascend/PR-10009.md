# vllm-ascend — D-side Partial-Group Caching for Hybrid Mamba Models

PR #10009 adds a minimal platform patch for KV consumer partial-group caching in hybrid Mamba models. The PR body explains that PD remote-prefill allocates D-side KV slots from computed cache hits before pulling KV from P-side. If the hybrid coordinator requires a common hit length across all cache groups, separately transferred Mamba state can force D-side hit rate to 0%.

The fix excludes the Mamba group from D-side KV cache hit-rate accounting while preserving the rest of the hybrid coordinator behavior.
