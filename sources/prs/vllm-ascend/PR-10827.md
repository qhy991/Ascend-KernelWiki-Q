# Fix Index Cache when DSA_CP is Enable

## Summary
The `self._get_indexcache_topk_indices` logic did not consider the DSA-CP (DeepSeek Attention Context Parallel) scenario. Under DSA-CP, `num_input_token` must be obtained from the DSA_CP context rather than assuming standard local lengths.
