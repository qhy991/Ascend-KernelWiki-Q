# vllm-ascend — Mooncake PP Layer Index and MTP Block Flattening Fix

PR #10202 fixes a Mooncake connector KV-transfer failure under pipeline-parallel configurations. The PR body explains that `get_num_layers()` returned per-stage layer count, while `extract_layer_index()` returned global layer numbers. With PP=2, MTP virtual-layer indices could collide with real base-layer indices.

The fix uses global hidden-layer count for `self.total_layers`, ensuring MTP virtual-layer indices start above all base model layer indices.
