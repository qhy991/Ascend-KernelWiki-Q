# vllm-ascend — DeepSeek-V4 Compressed Prefix-Cache Lookup Granularity

## What Changed

PR #10298 fixes compressed KV prefix-cache lookup for DeepSeek-V4. The PR body says cache write and lookup must use the same logical granularity: `block_size * compress_ratio`, not mixed physical and compressed block sizes.

The patch updates compressed cache manager logic, hybrid KV cache coordinator patches, and adds focused unit tests.

## Why It Matters

Compressed attention changes what a “block” means. If cache write uses compressed logical blocks but lookup receives physical-granularity hashes, prefix-cache hits can be wrong or truncated incorrectly. This is a semantic KV-cache bug, not a device attention math bug.
