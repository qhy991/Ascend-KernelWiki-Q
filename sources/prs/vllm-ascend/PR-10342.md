# vllm-ascend — Mooncake Connector Support for DeepSeek-V4 Hybrid KV

PR #10342 adds Mooncake connector support for DeepSeek-V4 and hybrid KV-cache scenarios. The PR body says prior Mooncake transfer logic assumed block ids could be translated with `block_id * block_len` and that prompt blocks were aligned the same way across all KV groups.

The fix adds per-block byte stride metadata to Mooncake handshake metadata and updates sender/receiver offset calculation to use tensor stride and element size, making mixed layouts such as attention KV, compressed KV, sliding-window KV, and Mamba/state groups safer.

Evidence read: GitHub PR body and file list; 5 files, +581/-94, merged 2026-06-17. Key files include `mooncake_connector.py`, `mooncake_layerwise_connector.py`, and KV offload tests.
