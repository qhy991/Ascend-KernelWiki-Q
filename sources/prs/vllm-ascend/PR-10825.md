# Fix low MTP acceptance rate for SFA+DSA_CP

## Summary
Fixes a degraded multi-token-prediction (MTP) / speculative-decoding acceptance rate on Ascend when `num_speculative_tokens > 1` for SFA models with `dsa_cp` enabled.

## Root Cause
In the SFA v1 attention-metadata builder, the `actual_seq_lengths_query` and `actual_seq_lengths_key` buffers handed to the lightning indexer shared the exact same memory buffer across all draft steps. With MTP > 1, the attention-metadata was overwritten, keeping only the last step. Rotary cos/sin were also fetched from an out-dated cache while drafting.

## Solution
Pre-allocate per-draft-step buffers (e.g. `spec_actual_seq_lengths_query`) so each step owns independent metadata.
Disable caching for rotary tables while drafting (`use_cache=False`) so tables are recomputed for each step.
