# vllm-ascend — AscendStore Grouped Hash Lookup Encoding Alignment

PR #10217 is a small correctness fix for AscendStore grouped hash lookup. The PR body explains that scheduler lookup RPC serialized block hashes as hex strings, while worker store path rehashed raw `BlockHash` bytes. After grouped rehashing, hashing hex-string bytes and raw bytes produced different grouped digests.

The fix updates `_block_hash_to_bytes()` to decode hex strings back to raw bytes before rehashing, while keeping UTF-8 fallback for non-hex strings.
